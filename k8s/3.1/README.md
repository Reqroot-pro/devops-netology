## 3. Обоснование

3.1. Ресурсы после вычета системных нужд (20%)

На каждой ноде резервируется 20% ресурсов под:
- kubelet, container runtime, kube-proxy
- системные демоны ОС
- агенты мониторинга и логирования

| Параметр | На одну ноду | На 4 ноды | На 3 ноды (при отказе 1) |
|----------|--------------|-----------|--------------------------|
| Доступно CPU | 6.4 ядра | 25.6 ядер | 19.2 ядра |
| Доступно RAM | 12.8 ГБ | 51.2 ГБ | 38.4 ГБ |

3.2. Проверка требований

-  **Нормальная работа (4 ноды)**: 25.6 CPU / 51.2 ГБ RAM > 17 CPU / 30.25 ГБ RAM — нагрузка помещается с запасом
-  **При отказе 1 ноды (3 ноды)**: 19.2 CPU / 38.4 ГБ RAM > 17 CPU / 30.25 ГБ RAM — принцип N+1 соблюдён
-  **Распределение стейтфул-сервисов**: 3 реплики БД и кеша корректно размещаются на разных нодах через `podAntiAffinity`
-  **Запас ресурсов**: остаётся ~2.2 CPU и ~8 ГБ RAM для пиковых нагрузок и горизонтального масштабирования
